# code-map: manifest

**Scope:** `packages/core/oas_core/manifest/` — `__init__.py`, `io.py`, `schema.py`
**Last verified:** 2026-05-23 — section `green-baseline`

## Purpose

Dataset manifest v1 — a typed, append-friendly description of a dataset version. Each manifest covers one **modality** (`ASR | TTS | LLM | S2S`) and is the unit of versioning, lineage, and reproducibility for everything downstream (training, eval, export).

Disk layout per dataset version:

```
<root>/
  manifest.json     # ManifestHeader (JSON, single object)
  samples.jsonl     # one Sample per line (canonical, streaming)
  samples.parquet   # optional columnar mirror (not yet generated by this module)
```

JSONL is canonical because it survives partial writes and is append/diff-friendly. Parquet is on-demand for analytics.

## Public API

- `Sample` (`schema.py:244`) — `Annotated[ASRSample | TTSSample | LLMSample | S2SSample, Field(discriminator="modality")]`. Discriminator is the `modality` field.
- `ManifestHeader` (`schema.py:264`) — header object: `schema_version`, `dataset_id`, `dataset_version`, `name`, `modality`, `license_default`, `stats`, plus optional metadata.
- `ManifestStats` (`schema.py:255`) — `num_samples`, `num_speakers`, `total_audio_s`, `by_split: dict`, `by_language: dict`.
- `ManifestWriter` (`io.py:38`) — context manager. Buffers stats in memory; flushes `manifest.json` only on `__exit__`/`close()`. Use for **bulk** writes (entire-manifest path).
- `ManifestReader` (`io.py:107`) — streaming iterator over samples; loads header eagerly. `load()` materializes the whole manifest into memory.
- `write_manifest(root, manifest)` (`io.py:132`) — one-shot dump of an in-memory `Manifest`.
- `read_manifest(root)` (`io.py:137`) — one-shot load.
- `update_sample(root, sample_id, patch)` (`io.py:141`) — apply a shallow patch by id; rewrites entire JSONL atomically via tmp + rename. Returns updated `Sample` or `None` if id missing.
- `init_manifest(root, header)` (`io.py:197`) — create an empty manifest dir: write header, ensure (do **not** truncate) `samples.jsonl`. Idempotent overwrite of `manifest.json`. Used by `apps/server/oas_server/routers/datasets.py:300` to persist a version row before any sample arrives.
- `append_sample(root, sample)` (`io.py:217`) — append one sample line and refresh header stats. Raises `FileNotFoundError` if the dir has no `manifest.json`. Used by `apps/server/oas_server/routers/datasets.py:348`.
- `export_json_schema()` (`io.py:272`) — JSON Schema for the discriminated `Sample` union; surfaced to web UI / SDK.

## Invariants

- `ManifestHeader.stats` is the source of truth for counts visible to the DB layer; routers read it after `append_sample` to refresh `DatasetVersion.num_samples` and `.total_audio_s` (see `apps/server/oas_server/routers/datasets.py:352-357`).
- One sample per JSONL line, each line `\n`-terminated. Empty lines tolerated by reader (`io.py:122`) but never produced.
- `num_speakers` is a **set cardinality**, not a counter. Recomputed by JSONL scan in both `_update_stats` (close-time, in `ManifestWriter`) and `append_sample` (every call).
- `Sample.split.value` is the dict key in `by_split` (Enum `Split` → string).
- Stats logic in `append_sample` (`io.py:243-268`) **must remain in sync** with `ManifestWriter._update_stats` (`io.py:77-94`). Any new counted field added to one must be added to both. (See "Gotchas" below — this is the most likely source of drift.)

## Concurrency model

- No locking. Single-writer assumed.
- `update_sample` (`io.py:141`) is atomic at the FS level via tmp + `replace()`.
- `append_sample` is non-atomic (append to JSONL, then rewrite header). Process killed between the two leaves JSONL ahead of header.stats — next read reconstructs samples correctly but stats are stale until the next successful append or a full scan.

## Error idioms

- `ManifestReader.__init__` raises `FileNotFoundError` if `manifest.json` is missing (`io.py:115`).
- `update_sample` raises `FileNotFoundError` if `samples.jsonl` is missing (`io.py:152`).
- `append_sample` raises `FileNotFoundError` if `manifest.json` is missing (`io.py:232`).
- No `panic`/`unwrap` equivalents — no bare `assert` on production paths, no `except: pass`. Sample validation errors propagate from Pydantic (`_SAMPLE_ADAPTER.validate_python`) to the caller; routers wrap to HTTP 400 (`apps/server/oas_server/routers/datasets.py:346-347`).

## Gotchas

- **Stats parity** between `ManifestWriter._update_stats` and `append_sample` is hand-maintained. If you add `by_duration_bucket` to stats, you must touch both. A shared helper would be cleaner; deliberately deferred (would have widened T1 beyond the bugfix). Track here so future-you doesn't introduce drift.
- **`init_manifest` re-init footgun:** overwrites `manifest.json` (including `schema_version`) but never touches `samples.jsonl`. If a caller re-inits with a header whose `schema_version` differs from what existing samples were written under, the manifest becomes internally inconsistent. No caller does this today; gate at the route layer rather than here if it ever becomes a real concern.
- **`update_sample` rewrites the entire JSONL** for one sample change. Documented as "good up to ~1M samples" at `io.py:147`. The same scale applies to `append_sample`'s `num_speakers` recompute scan.

## Open questions

- Should `init_manifest` accept a `truncate: bool = False` flag so re-init for a fresh empty version is explicit? Currently the dataset router enforces "directory empty" at the route layer (`routers/datasets.py:291`) — keeping the truncate decision out of the io layer. Revisit if the second caller appears.
- Parquet mirror generation lives nowhere yet — listed in the file docstring at `io.py:11` but no function exists. Surface as a P3 candidate when TTS/ASR analytics dashboards land.
